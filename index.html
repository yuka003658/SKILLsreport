Section 01

Anthropicとは何か

Anthropicは、2021年にアメリカで設立されたAI企業です。ChatGPTで有名なOpenAIの元メンバーたちが「AIの安全性をもっと真剣に考えたい」という思いから独立し、設立しました。

「AIを速く作ること」より「AIを安全に作ること」を最優先にした会社、それがAnthropicです。

車メーカーで例えると、OpenAIがトヨタ、AnthropicがHonda、GoogleがNissanのような関係です。それぞれ別の会社・別のシステムで動いているため、一方のツールやファイルが他方で使えることはありません。

Section 02

ArtifactとSkillの違い

Claudeを使っていると「Artifact」と「Skill」という二つの概念が出てきます。どちらも重要ですが、役割がまったく異なります。

Artifact（成果物）
作られたもの
Claudeが実際に生成して「渡す」ファイルのこと。コード、HTMLページ、Word文書、Markdownレポートなど。ユーザーが見たり、ダウンロードしたりできる。
Skill（スキル）
作り方マニュアル
Claudeが作業前に内部で読む「手順書」のこと。サーバー上のファイルとして存在し、ユーザーには見えない。Claudeの出力品質を高めるための裏側の仕組み。

フローで見る違い

Input
あなたのリクエスト
→
Internal
Skillを読む（非表示）
→
Process
Claudeが考える・処理する
→
Output
Artifactが生成される

Section 03

Skillの仕組みと使い方

SkillはAnthropicのサーバー上の /mnt/skills/ というフォルダに置かれたテキストファイルです。Claudeは「Wordファイルを作る必要があるな」と判断したとき、自分でそのファイルを読みに行きます。

docxスキル
Word文書
Word文書の正しい作り方・注意点が書かれている
pptxスキル
スライド
PowerPointスライドの作り方が書かれている
frontend-designスキル
Web UI
質の高いWeb画面の設計ルールが書かれている

一般ユーザーはSkillを追加できるか？

できません。Skillはサーバー側の設定であり、一般ユーザーが追加・編集できるものではありません。ただし、似た効果を得る方法はあります。

方法 できること 難易度
カスタム指示（claude.ai設定） 毎回の会話の前提を設定できる ★☆☆ 簡単
プロジェクト機能 案件・用途ごとに設定・資料を分けて管理 ★★☆ 普通
APIでシステムプロンプト Skillに近い詳細な指示を埋め込める ★★★ 開発者向け

広告代理店のCDとして使うなら、「プロジェクト機能」でブランド戦略用・キャンペーン用などを分けて作るのが最初の一歩です。

Section 04

ChatGPT・Geminiとの仕組みの違い

「事前に読ませる指示・知識で出力品質を上げる」という考え方はどのAIも同じです。ただし実装方法はそれぞれ異なります。

AI 類似機能の名前 仕組みの特徴
Claude（Anthropic） Skill サーバー上の実ファイルをClaudeが自分で読みに行く。エンジニアリング的アプローチ。
ChatGPT（OpenAI） GPTs・カスタムインストラクション システムプロンプトに指示を埋め込む方式が主流。
Gemini（Google） Gems 同様にプロンプト埋め込み方式が中心。

重要なのは、これらは会社ごとに完全に独立したシステムということです。AnthropicのSkillはChatGPTでは使えませんし、ChatGPTのGPTsはClaudeでは動きません。トヨタの整備マニュアルをホンダの工場で使えないのと同じです。

ユーザーに見えないという点は、どのAIでも共通しています。
